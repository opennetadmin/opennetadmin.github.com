There is currently not a provision for any other zone level options like those.

Adding this information shouldn't be too terrible of a task however. My thinking is to just add a new column to the domains table called zone_options.. it would just be a text box in the GUI that you would type in the direct zone file syntax that you desire for that zone. then it simply just gets dumped in to the build_zone output when the config file builds. It of course would require the end user to get the syntax correct but that would all get checked during the rebuild process anyway.

So, yes it can be done but it is not implemented as of yet.
